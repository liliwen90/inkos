## 题材禁忌

- 开局就什么都会：种地、盖房、冶铁、医术样样精通
- 资源无限且获取零成本——想要什么就能找到
- 种田=安逸无冲突，完全没有外部威胁和内部矛盾
- 技术发展无视当地材料和人力条件
- 所有投靠的人都是忠心耿耿的好人
- 纯粹的清单式建设记录：建了A→又建了B→还建了C
- 忽视管理难度——几个人的村子和几千人的城镇管理方式一样
- 所有问题都能用"种田"解决（种出粮食→换钱→买兵→打赢→继续种田）

## 经营规则

- 资源有约束：原料、人力、时间、运输成本都是硬限制
- 技术发展有前置条件：想烧砖要先找到合适的黏土和燃料，想炼铁要先建窑
- 经济体系要合理：供需关系、贸易行情、货币稳定性都影响经营
- 人口是资源也是负担：来的人需要吃饭、需要住处、需要工作、需要管理
- 规模升级伴随管理难度跃升：10人靠威望→100人靠制度→1000人靠组织架构
- 季节和天气对农业有实质影响——不是播种→等待→丰收这么简单
- 外部势力（官府、匪帮、商帮、邻居）对领地发展有真实影响

## 语言铁律

- 建设过程用具体的材料和步骤，不用"很快就建好了"跳过
- ✗"他们花了三天时间建起了一座水车" → ✓"第一天在上游找到了合适弯曲度的老柏木，第二天的榫卯接合失败了两次——水流的冲击力比他预想的大"
- 收获的喜悦用具体的数字和实物传达：✗"丰收了" → ✓"六亩地打出了十二石谷子，比去年多了三成。够吃到明年开春，还能换两匹粗布"
- 人来投靠的过程要有代价和考验——不是所有人都值得信任
- 管理问题用具体的冲突展现：分配不公、偷懒、派系、外来者和本地人的矛盾
- 日常感是种田文的魅力——早起鸡鸣、傍晚炊烟、修补工具的声音比战斗更有沉浸感

## 对话铁律

- 种田文的对话是生活化的：讨论天气、收成、物价、邻里关系
- 手下人的汇报要具体：✗"一切顺利" → ✓"北边那块地的麦子倒了一片，估计是昨晚的大风。有三户佃户来问能不能缓交租子"
- 谈判/交易对话要有利益博弈：买卖双方都在试探底线
- 技术指导对话要有专业感但不枯燥：用比喻和实操代替教科书
- 投靠/招募对话要现实：人来是因为有好处——工钱、安全、土地，不是因为慕名而来
- 管理冲突的对话要有温度：不是冰冷的制度宣读，是带着人情味的处理

## 反AI写作模式

- 禁止建设流水账：✗"他建了磨坊，又建了铁匠铺，还修了一条路" → 选一个建设过程展开写透
- 禁止"金手指式科技树"：不是带着全套现代工程知识穿越的百科全书
- 禁止所有外来者都心服口服——人的忠诚需要理由和时间
- 禁止"一个月后已经初具规模"的跳跃——时间跳跃必须交代产生了什么问题
- 禁止忽略天灾、虫害、疫病、旱涝——这些才是农业社会的真实敌人
- 日常不等于灌水：每个日常章节都要推进一条线（人际、经济、技术、外部关系）

## 章节钩子

- 建设章结尾：工程完成但发现选址有问题 / 新技术成功但被外人注意到了 / 缺少关键材料需要外出采购
- 经营章结尾：第一批产出成功但价格被商帮压低 / 人口增长带来了新的管理问题 / 有人偷学了核心技术
- 社交章结尾：外部势力表达了"合作"意向——友善的表面下有觊觎 / 官府来人了
- 危机章结尾：度过了眼前的危机但暴露了体系的弱点 / 为解决危机付出的代价开始显现

## 叙事指导

种田文的核心爽感是"从无到有"——把一片荒地变成家园的成就感。
进步不需要快，但必须可见——每完成一件事都让"家"更完整。

建设不是目的，人的生活改善才是目的。
要写吃到新收获粮食时的满足、第一次喝上干净水时的幸福——这些小幸福是种田文的灵魂。

危机不一定来自外部敌人——自然灾害、内部分裂、资源枯竭也是有力的剧情引擎。
好的种田文在"建设的快乐"和"维持的艰难"之间找到平衡。
纯建设会平淡，纯危机会压抑——二者的交替创造节奏。

人际关系是种田文的核心维度：谁值得信任、谁有野心、谁是真正的伙伴。
领导不是天赋——是在一次次决策中积累的威信和经验。
